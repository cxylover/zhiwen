## 目标
带你把本项目认证（登录/注册）模块从“入口 → 业务 → 安全 → 存储 → 外部依赖”完整走一遍，明确每一步做什么、用到了哪些 Bean、关键配置项是什么。

## 我将按以下顺序分析
1. **认证模块边界与包结构**：哪些包属于 auth，哪些属于 user/profile 等。
2. **配置与 Bean 组装**（你刚看的 `AuthProperties` / `AuthConfiguration`）：
   - `PasswordEncoder` 如何创建（BCrypt 强度来自配置）
   - `JwtEncoder` / `JwtDecoder` 如何从 PEM key 构建（Nimbus + JWKSet）
   - 这些 Bean 被哪些地方注入
3. **安全过滤链**（Spring Security）：
   - `SecurityConfig` 里 HTTP 安全怎么配置（哪些路径放行、哪些需要鉴权）
   - 资源服务器模式如何校验 Bearer JWT（`spring-boot-starter-oauth2-resource-server`）
   - JWT claim（issuer/kid）与 key 的对应关系
4. **登录/注册接口**：
   - 找到 `AuthController` / `UserController`（或类似命名）
   - 请求 DTO / 响应 DTO
   - 注册：密码如何 hash、用户如何入库、是否校验验证码
   - 登录：如何校验密码、如何生成 access/refresh token、token 如何返回
5. **验证码**（如果有）：
   - 发送与校验流程（Redis 存储、限流/重试、每日上限）
6. **刷新 token / 退出登录**（如果有）：
   - refresh token 存储策略（纯 JWT / 服务端存储 / 黑名单）
   - token 失效与撤销
7. **数据层**：
   - 用户表结构 / mapper / repository
   - 唯一性约束（邮箱/手机号/用户名）与异常处理
8. **端到端数据流总结**：
   - 登录：HTTP → Service → PasswordEncoder → JwtEncoder → Response
   - 鉴权访问：HTTP → Security Filter → JwtDecoder → Authentication → Controller
